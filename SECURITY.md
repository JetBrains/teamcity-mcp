# Security Policy

We do our best to make sure the TeamCity MCP Server plugin is free of security vulnerabilities.
We are very eager and grateful to hear about any issues you find.

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Report vulnerabilities through one of the following channels:

- **JetBrains issue tracker** — create an issue at https://youtrack.jetbrains.com/newIssue?project=TW
  and set the issue type to **Security Problem** to limit visibility.
- **Email** — send details to [security@jetbrains.com](mailto:security@jetbrains.com),
  encrypted with our [PGP key](https://www.jetbrains.com/privacy-security/) if needed.

The report should include a description of the vulnerability and a working proof of concept.
Please use your own data when demonstrating the issue and limit the impact to the minimum
required to prove the concept.

For more information on how our responsible disclosure process works, see the
[JetBrains Coordinated Disclosure Policy](https://www.jetbrains.com/legal/docs/terms/coordinated-disclosure/).

## Response Timeline

| Stage | Target |
| --- | --- |
| Acknowledgment | Within 24 hours |
| Triage & planned course of action | Within 24 hours of acknowledgment |
| Fix shipped | With the next plugin release or bug-fix update |
| Public disclosure | Announced on [JetBrains Blog](https://blog.jetbrains.com) after customers have had time to update |
| Security bulletin | Included in the quarterly [JetBrains Security Bulletin](https://www.jetbrains.com/privacy-security/issues-fixed/) |

## Bug Bounty

We don't have a formal bug bounty program, but depending on the severity of the issue
we may reward reporters based on the CVSS score. Rewards are determined at our discretion
and may include financial rewards, swag, or free JetBrains product licenses.

## Supported Versions

We provide security fixes for the latest released version of the plugin only.
We recommend always running the most recent release.
